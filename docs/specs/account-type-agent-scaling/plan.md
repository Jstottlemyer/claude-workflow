# Implementation Plan — account-type-agent-scaling

Generated by `/plan` in autonomous mode. Synthesizes seven design personas (api, data-model, ux, scalability, security, integration, wave-sequencer) and resolves the spec-review blockers (B1–B10) and risk register (R1–R14) into a concrete, ordered task breakdown.

The plan is deliberately small in surface area: this feature adds **one new resolver process**, **one new config file**, and **a banner** to existing gate dispatch — but it sits in the pre-flight of every gate, so the design prioritizes *fail-safe under degradation* over *features*.

### Three load-bearing decisions

1. **Resolver is Python, not bash.** macOS bash 3.2 cannot reliably parse JSON, and three writers (install.sh, runtime, "tell Claude") each touching the same JSON file is too fragile for hand-rolled `sed/awk`. A thin bash wrapper (`scripts/resolve-personas.sh`) shells out to `scripts/resolve-personas/resolver.py`. Python 3.9+ is already required by MonsterFlow. (Resolves I1, R3.)

2. **Personas are disk-discovered, not hardcoded.** Seed lists become *priority orderings* — names from a seed that exist on disk are preferred; the on-disk remainder fills lexically. This makes the feature roster-fork-safe (R9) and removes the B1 "/plan cites missing `risk` persona" class of bug structurally — the resolver cannot emit a name that has no file.

3. **Resolver output is a structured JSON line, not just persona names.** AC #9's `Selected: ... | Dropped: ...` banner needs both halves; the gate also needs Codex eligibility, fallback reason, and rankings freshness for the new banner format. Bash callers receive newline-separated names on stdout for back-compat (`--names-only` mode), but `--json` is the canonical contract. (Resolves B8.)

### Resolution algorithm (canonical pseudocode — locked in `docs/budget.md`)

```python
def resolve(gate, config, rankings, disk_personas, codex_state):
    budget = clamp(config.agent_budget or DEFAULT_BUDGET, 1, 8)
    pins = [p for p in (config.persona_pins.get(gate) or []) if p in disk_personas]
    pins = pins[:budget]                                  # FIFO truncation; I6
    selected = list(pins)                                 # pins consume budget slots; B3 reading (a)

    # Rankings: gate-matching, qualifying rows only
    ranked = sorted(
        [r for r in rankings
         if r.gate == gate
         and r.persona in disk_personas
         and r.persona not in selected
         and qualifies(r)],                               # see "qualifies" below; B2
        key=lambda r: (r.score, -r.insufficient_sample_weight),
        reverse=True,
    )
    for r in ranked:
        if len(selected) >= budget: break
        selected.append(r.persona)

    # Seed fill: priority order, then lexical
    seed_priority = SEEDS[gate]                           # priority list, not exhaustive
    seed_pool = [p for p in seed_priority if p in disk_personas and p not in selected]
    seed_pool += sorted(p for p in disk_personas
                        if p not in seed_priority and p not in selected)
    for p in seed_pool:
        if len(selected) >= budget: break
        selected.append(p)

    # Codex is additive, gate-eligible, never counted against budget
    codex_eligible = (
        gate in CODEX_GATES                               # spec-review, check; NOT plan in v1 — B6
        and codex_state == "authenticated"
    )
    if codex_eligible:
        selected.append("codex-adversary")

    dropped = [p for p in disk_personas if p not in selected]
    return SelectionResult(selected, dropped, fallback_reason, ...)
```

`qualifies(r)`: rows with `insufficient_sample == true` are *included with deprioritization weight 0.5x* rather than excluded — prevents low-traffic gates (R13) from starving rankings selection while still preferring well-sampled personas.

## 2. Key Design Decisions (with rationale)

| # | Decision | Why | Resolves |
|---|---------|-----|----------|
| D1 | Resolver implementation language: **Python 3.9+** with thin bash wrapper | bash 3.2 JSON parsing is brittle; three writers need shared validation lib | I1, I3, R3 |
| D2 | Persona discovery: **disk-walk `personas/<gate>/*.md` at every invocation**; seeds are priority hints, not roster definitions | Adopter forks rename/add/drop personas; hardcoded seeds break first install | B1, R9, I4 |
| D3 | Resolver contract: **`--json` mode is canonical**; `--names-only` for back-compat | Banner needs Selected + Dropped + Codex eligibility + fallback reason | B8 |
| D4 | Algorithm reading of B3: **(a) pins-consume-budget, then rankings, then seed** | Most predictable; pins are the explicit user voice; rankings are data-driven; seed is structural fallback | B3 |
| D5 | Pin truncation: **FIFO (list order)** when pins > budget | Predictable; matches user's mental model from install Q&A order | I6 |
| D6 | Recovery options on resolver error: **(1) reconfigure now, (2) continue with seed, (3) abort gate**. Drop "disable budget for this run." | Option (3) in spec contradicted AC #7 by silently restoring full roster | B4 |
| D7 | Codex per-gate: **`/spec-review` and `/check` only in v1**; `/plan` opts out | `/plan` has no Codex phase today; schema/metrics already reflect that; structural symmetry deferred | B6 |
| D8 | Default-no-config behavior: **resolver outputs full disk roster in priority+lexical order**; banner is the one acknowledged change | Preserves existing dispatch shape; AC #1 reworded to "no persona-set change" | B5 |
| D9 | `agent_budget` defaults: **Pro=3, non-Pro=6**, ceiling=8 | Asymmetric is the point — Pro gets tighter budget to avoid rate limits | I8 |
| D10 | `agent_budget` validation: **floor=1, ceiling=8**; write-time rejects ≤0 and warn-caps >8; runtime mirrors | Defense in depth; unsigned int isn't enforceable in JSON | spec |
| D11 | `persona_pins` schema: **list per gate** (singular pin allowed; AC #3 reworded to "first pin or seed[0]") | Future-proofs for >1 pin per gate; install Q&A starts at 1 pin and is extendable | I7 |
| D12 | Codex auth check: **`MONSTERFLOW_CODEX_AVAILABLE` env-var cache**, set by autorun (or first gate in interactive); `command -v codex` short-circuit; `timeout 2` on `codex login status` | Per-gate calls add latency, may emit telemetry, may hang on SSO | I2, R12 |
| D13 | Config writes: **`mktemp + json.load validation + mv -f`**, with `.bak` rotation; `schema_version: 1`; `$HOME` not `~` in code | Atomic; rollback path; future migrations | I17, R3 |
| D14 | Config reads during autorun: **snapshot at autorun start to `dashboard/data/budget-snapshot-<run-id>.json`**, exported via `MONSTERFLOW_BUDGET_SNAPSHOT` | Reconfigure mid-autorun cannot produce mid-pipeline inconsistency | R14 |
| D15 | Bypass: **`MONSTERFLOW_RESOLVE_PERSONAS_BYPASS=1`** env var skips resolver entirely; gate dispatches full disk roster | Documented escape hatch when resolver itself is the bug | R1 |
| D16 | Empty-stdout guard: **gate caller treats `len(selected_claude_personas) < 1` as hard error** (not just non-zero exit) | Prevents zero-persona dispatch from a "successful" resolver call | R1, I18 |
| D17 | Fallback observability: **resolver appends one row per invocation to `dashboard/data/budget-events.jsonl`** (decision record) | Headless degradation surfaces in `/wrap-quick` and autorun report | I13, R2 |
| D18 | Rankings freshness: **stderr warning + banner annotation** when `persona-rankings.jsonl` mtime > 30 days; hard re-emphasis at >90 | Prevents silent reliance on stale rankings | R5 |
| D19 | `participation.jsonl` schema v2: **add `dispatch_pool[]` and `roster_size_total`** (optional in schema v1 readers) | Phase 1c persona drift becomes longitudinal-comparable; budget-dropped ≠ silent | B10, R10 |
| D20 | Defaults source-of-truth: **`scripts/resolve-personas/defaults.json` + `seeds.json`** read by resolver, install.sh, and `tests/test-defaults-consistency.sh` | Single canonical source; doc/code drift impossible | O7, R6 |
| D21 | Reconfigure flag: **`install.sh --reconfigure[=budget|pins|all]`** (default: `all`). Keep `--reconfigure-budget` as v1 alias | Avoids future flag explosion | O8 |
| D22 | "Tell Claude to reconfigure": **invokes `commands/_prompts/budget-qa.md`** — same Q&A install.sh uses, no parallel implementation | One Q&A module, three callers (install, --reconfigure, Claude) | I3, I10 |
| D23 | Linux behavior: **install.sh skips Q&A, writes nothing, prints `[install] budget Q&A is macOS-only in v1; using full-roster default`** | macOS-only is explicit; Linux adopters not silently broken | I14 |
| D24 | Idempotent install: **schema_version match → skip Q&A**; mismatch → migration prompt | install.sh runs repeatedly on upgrades | R7 |
| D25 | Banner is **a stable, parseable contract** documented in `docs/budget.md` | Prevents accidental contract via downstream parsers | R8 |
| D26 | token-economics "live" precondition: **≥1 row in `dashboard/data/persona-rankings.jsonl` for the gate being dispatched** is the qualifying signal; absent rows = silent seed-mode (warning at install if directory exists but file is empty) | Concrete, checkable precondition | B9 |
